## Objective
Verify: Include `*.txt` + Exclude `Text.txt` → all `.txt` backed up EXCEPT `Text.txt`/`text.txt`
(case-insensitive matching), `.doc` files not included at all (not matched by the inclusion).

## Fixtures used
- Source: `linux-src` **PM-2** (`Linux_16.84`) — **note:** deliberately using Linux (ext4, a
  case-SENSITIVE filesystem) rather than Windows for this fixture — NTFS is case-insensitive at
  the filesystem level, so `Text.txt` and `text.txt` cannot coexist as two distinct files on
  Windows. Using Linux lets both files genuinely exist, so the test actually exercises whether
  NBR's *rule-matching* is case-insensitive (per spec), rather than being masked by the
  filesystem's own case-folding.
- Test data: `/TestData_ForFLB/IncludeExclude/TxtOverruleA/` — `Text.txt`, `text.txt`,
  `other.txt`, `keep.doc` (to be seeded)
- Repository: **Onboard** (`BACKUP_REPOSITORY-2`)
- Job build: **R4c** + Inclusion (`*.txt`) + Exclusion (`Text.txt`)
