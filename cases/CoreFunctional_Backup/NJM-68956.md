## Objective
Verify FLB preserves file/folder **permissions (ACLs)**: with ACL preservation enabled, backed-up
folders recover to a clean location with the **same ACLs** as source (spec: Options ACL =
"Back up folder and file permissions").

## Fixtures used
- Source: windows-src **PM-3** (or PM-2) — folders with known/explicit ACLs
- Repository: **Onboard** (`BACKUP_REPOSITORY-2`)
- Job build: **R4c**; Job name: `AUTO_FLB_NJM-68956`

## Preconditions
- [ ] Source present & OK — **R1**; Onboard repo online — **R3**
- [ ] **Setup (remoting):** set explicit ACLs on the source folders (`icacls` on Windows / `setfacl`
      on Linux); record the ACLs.
- [ ] **BLOCKED guard:** if ACLs can't be set/compared → PARTIAL/BLOCKED, note why.

## Steps
### Step 1 — Build — R4c: `sourceVid=PM-3`, folder mapping (ACL-tagged folders), BackupOptionsBuilder `options.accessControlList="FOLDER_AND_FILE_PERMISSIONS"`, `targetStorageVid=BACKUP_REPOSITORY-2`, name `AUTO_FLB_NJM-68956` → `saveJob`.
### Step 2 — Run — R5 (`runType:ALL`) · **Expect:** `lrState:OK`.
### Step 3 — Recover to CLEAN custom location — FLR `recover` **EXPORT to a CIFS/NFS share** (a clean path). ⚠ Do NOT use "Recovery to original location" (overwrites source — safety fence).
### Step 4 — Compare ACLs — remoting: read recovered ACLs (`icacls`/`getfacl`) and compare to the recorded source ACLs — must match (no perms lost/altered).

## Cleanup
- On PASS — **R9** `remove [<jobId>, false]` + clean the export path. On FAIL — keep + record.

## Expected final verdict
PASS if `lrState:OK` and recovered folders' ACLs match the source ACLs exactly.
