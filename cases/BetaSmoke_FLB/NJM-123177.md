## Objective
Verify an FLB recovery point copied (via Backup Copy Job) into an immutable Object-Lock-enabled
Amazon S3 repository can be recovered to a CIFS share, matching source content, with the S3
Object Lock remaining intact.

## Fixtures used
- Source: an existing FLB backup object on Onboard (e.g. from a prior `AUTO_FLB_*` run), or a
  fresh single-file backup built for this TC
- Repository: **Amazon_Immutable** (`BACKUP_REPOSITORY-10`) ‚Äî added 2026-07-08
- Job build: **R4d** ‚Äî `test-data/job-templates/backup_copy_job.template.json`
  (`hvType:"VMWARE"` fixed, `sourceVid=BACKUP_OBJECT-<id>`, `targetStorageVid=BACKUP_REPOSITORY-10`)
  + `options.retentionPolicy.retentionMode="RULESET"` for immutability (introspect shape first)
- Job name: `AUTO_FLB_NJM-123177`

## Preconditions
- [ ] Source `BACKUP_OBJECT` present & accessible ‚Äî **R1**-equivalent; `Amazon_Immutable` repo
  online ‚Äî **R3**.
- [ ] **BLOCKED guard:** if `saveJob` rejects the immutable-retention shape, or the resulting
  savepoint does not show `isImmutableSavepoint:true`, stop and report BLOCKED/FAIL ‚Äî do not infer
  immutability from the repo's Object Lock capability alone.

## Steps
### Step 1 ‚Äî Introspect ‚Äî `mcp__nbr__describe_method nbr-84 JobManagement saveJob` (confirm
  immutable-retention field shape).
### Step 2 ‚Äî Build ‚Äî R4d: source `BACKUP_OBJECT-<id>`, `targetStorageVid=BACKUP_REPOSITORY-10`,
  `retentionPolicy.retentionMode="RULESET"`, name `AUTO_FLB_NJM-123177` ‚Üí `saveJob`.
### Step 3 ‚Äî Run ‚Äî R5 (`runType:ALL`).
### Step 4 ‚Äî Poll ‚Äî R6 ¬∑ **Expect:** `lrState:OK`, `lrVmOk:1`.
### Step 5 ‚Äî Verify immutability ‚Äî `JobManagement.getSavepoints` on the new backup object;
  **Expect:** `isImmutableSavepoint:true`.
### Step 6 ‚Äî Recover ‚Äî FLR `createSession`/`list`, then `recover` with `recoveryType:"EXPORT"` to
  a CIFS share; SHA-256 vs the source manifest.

## Cleanup
- On PASS ‚Äî **R9** `remove [<jobId>, false]` (an immutable savepoint may resist removal until its
  lock expires ‚Äî expected).
- On FAIL/BLOCKED ‚Äî leave artifacts; record job name/id.

## Expected final verdict
PASS if `lrState:OK`, the savepoint shows `isImmutableSavepoint:true`, and the CIFS-exported copy
matches source checksums. FAIL (not BLOCKED) if the job runs but immutability doesn't actually
engage ‚Äî that would be a real product gap, not an environment gap.
