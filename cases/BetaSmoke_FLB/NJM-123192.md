## Objective
Verify FLB recovery points stored across Local (Linux), S3, and Azure Blob immutable
repositories can each be recovered to a CIFS share, matching source content, with all three
repositories' lock statuses remaining intact.

## Fixtures used
- Repositories: **Local-Immutable** (`BACKUP_REPOSITORY-15`, Local leg), **Amazon_Immutable**
  (`BACKUP_REPOSITORY-10`), **Azure_Immutable** (`BACKUP_REPOSITORY-12`)
- Job build: **R4c**, one job per repo, each with `options.retentionPolicy.retentionMode="RULESET"`
  (+ immutable count per the confirmed shape — introspect first)
- Job names: `AUTO_FLB_NJM-123192_local`, `AUTO_FLB_NJM-123192_s3`, `AUTO_FLB_NJM-123192_azure`

## Preconditions
- [ ] Source present & OK — **R1**; all three repos online — **R3**.
- [ ] **BLOCKED guard:** if `saveJob` rejects the immutable-retention shape on any leg, or a
  savepoint doesn't show `isImmutableSavepoint:true`, report that leg FAIL — do not infer
  immutability from the repo's Object Lock capability alone.

## Steps
### Step 1 — Introspect — `mcp__nbr__describe_method nbr-84 JobManagement saveJob` (confirm the
  immutable-retention field shape once this session).
### Step 2 — For each of the three repos (Local-Immutable, Amazon_Immutable, Azure_Immutable): R4c
  build with `retentionPolicy.retentionMode="RULESET"`, targeting that repo, distinct job name → `saveJob`.
### Step 3 — Run each — R5 (`runType:ALL`).
### Step 4 — Poll each — R6 · **Expect:** `lrState:OK`.
### Step 5 — Verify immutability per leg — `getSavepoints`; **Expect:** `isImmutableSavepoint:true`
  on all three.
### Step 6 — Recover each — FLR `recover` with `recoveryType:"EXPORT"` to a CIFS share; SHA-256 vs
  manifest, per leg.

## Cleanup
- On PASS (per leg) — **R9** `remove [<jobId>, false]` (immutable savepoints may resist removal
  until their lock expires — expected). On FAIL/BLOCKED — leave artifacts; record job name/id.

## Expected final verdict
PASS if all three legs reach `lrState:OK`, each shows `isImmutableSavepoint:true`, and each
CIFS-exported copy matches checksums. PARTIAL if some legs pass and others don't; FAIL (not
BLOCKED) if immutability doesn't actually engage on a leg whose repo is otherwise healthy.
