# NJM-123510 — [FLB v1] FLR from FLB - Functional - Verify Recovery from an Encrypted Backup

- **Source TC:** NJM-123510 (Xray, FLB/Share, Blocker P1, flb-v1)
- **Feature:** File-Level Backup (physical machine)
- **Appliance:** nbr-84 (FLB) — see `test-data/environment.md`
- **Metadata:** testcase_id=NJM-123510, author=tri.ton, date=2026-07-08, product=FLB, status=GENERATED
- **Feasibility:** ⚠️ PARTIAL — building an **encrypted** job is feasible via R4c's
  BackupOptionsBuilder (`options.backupEncryptionMode` + `options.encryptionPasswordId` in
  `flb_job.template.json`), but the TC's final step is **"Recover to original location"**, which
  `recipes/file-backup-recipes.md` R7 explicitly flags as **blocked by framework safety**
  (`recoveryType:"ORIGINAL"`+`OVERWRITE` overwrites the source). Substitute: verify recovery via
  FLR EXPORT to a CIFS share instead of original-location overwrite.

## Objective
Verify a File-Level Backup created with **encryption enabled** can be recovered (via a
non-destructive EXPORT, substituting for the TC's original-location step) with all files —
including nested folder structures — restored and matching source content/checksums exactly.
